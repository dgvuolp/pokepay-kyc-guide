## PokePay 身份認證分兩級：L1 對應虛擬卡，L2 對應實體卡

PokePay 的 KYC 是分級的，這點很多人沒意識到，結果開虛擬卡時把實體卡那一套資料都備齊了，其實完全沒必要。

**KYC Level 1（虛擬卡專用）**

只需要三樣東西：法定姓名、身份證正反面照片、人臉識別。整個流程在 APP 內完成，正常幾分鐘就能通過。L1 對應的是 5347、559666、404038、556150、441112 這些虛擬卡卡段，開完就能線上消費。

**KYC Level 2（實體卡專用）**

實體卡因為涉及 ATM 取現、線下 POS 刷卡，反洗錢要求更嚴格，需要完成 L2 級別認證。L2 在 L1 的基礎上會要求更完整的身份資訊（部分情況可能需要地址證明，如水電費帳單、銀行對帳單等）。L2 通過後才能申請那張 88U 開卡費的港幣 Visa 實體卡。

## PokePay 身份認證詳細操作步驟

以下是 L1 級別的完整流程，L2 在此基礎上增加資料項即可：

1. **進入認證入口**：登入 PokePay APP 或網頁端，在側邊導航找到「身份認證」中心，點擊進入 KYC Level 1。
2. **填寫個人資訊**：填寫法定姓名——**上面填名字（First Name），下面填姓氏（Last Name）**，順序別搞反。證件類型支援身份證和護照，國內用戶直接選身份證就行，沒必要特意去辦護照。
3. **上傳證件**：拍身份證正反面，確保照片清晰、四角完整、無反光無遮擋。建議在光線均勻的環境下拍，別用閃光燈。
4. **人臉識別**：按提示完成活體檢測，眨眼、轉頭之類的動作跟著做就行。
5. **等待審核**：通常幾分鐘到幾小時內出結果，少數情況可能需要人工複核。

> 官方 FAQ 明確提醒：**填寫的資訊請與上傳的證件完全保持一致，否則 KYC 無法通過**。姓名填寫請與證件保持一致；使用護照實名時，姓名一般填英文。建議使用谷歌瀏覽器並開啟相機權限。

## PokePay 身份認證失敗的常見原因與對策

這是搜索「PokePay 身份認證」的人最常卡住的點。把高頻翻車原因列一下，對著自查：

| 失敗原因 | 具體表現 | 解決方法 |
| --- | --- | --- |
| 姓名順序填反 | First Name 填了姓，Last Name 填了名 | 嚴格按「名字在上、姓氏在下」填寫，與證件拼音一致 |
| 證件資訊不一致 | 填的姓名與上傳的身份證對不上 | 重新核對，確保填寫內容與證件一字不差 |
| 證件照片不清晰 | 反光、遮擋、四角缺失、模糊 | 重新拍攝，光線均勻、平鋪、四角完整入鏡 |
| 相機權限未開 | 人臉識別環節黑屏或無法啟動 | 在瀏覽器或 APP 設置中開啟相機權限，建議用 Chrome |
| 護照填中文姓名 | 用護照實名卻填中文名 | 護照實名時姓名一般填英文，與護照資料頁一致 |
| 活體檢測失敗 | 動作識別不通過 | 光線充足、正對鏡頭、按提示緩慢完成動作 |

如果以上都檢查過還是過不了，直接聯繫 PokePay 官方客服，提供帳號和截圖，一般能協助定位問題。

KYC 過了不等於能開卡，**帳戶得先有錢**。PokePay 只接受加密貨幣充值，支援 USDT（TRC20/ERC20/Polygon）、USDC、BTC、ETH。流程是：

3. 從交易所提幣到該地址，**充值手續費 1%**（充 100U 到帳 99U），一般 15 分鐘左右到帳。
4. 首次充值建議 **25–30 USDT**，覆蓋開卡費 + 手續費綽綽有餘。

到帳後餘額自動轉成港幣或美元，就能去「卡片」頁面申請開卡了。

## PokePay 身份認證與使用注意事項

最後把幾個容易踩的坑整理一下，認證和使用時對著避開：

1. **IP 問題**：訂閱 ChatGPT Plus 時，消費 IP 最好與註冊 IP 地區一致，否則可能觸發風控。
2. **CVV 別反覆試**：多次輸錯 CVV 或有效期會鎖卡，需要聯繫客服解鎖，別抱著僥倖心理亂試。
3. **實體卡 PIN 碼**：同理，PIN 碼多次輸錯也會鎖定，記不住就找客服重置，別硬試。
4. **證件資訊一致性**：KYC 填的姓名、證件號必須與上傳證件一字不差，這是失敗率最高的環節。
5. **相機權限**：用 Chrome 瀏覽器並提前在設置裡開啟相機權限，否則人臉識別環節直接卡住。
6. **小額消費不划算**：低於 10 USD 的消費手續費占比過高，建議合併消費或選無小額費的 5347 卡段。
7. **退款時間**：普通商家退款 3–7 個工作日，跨境交易退款可能 7–30 個工作日，爭議退款最長 90 天，提前做好資金周轉預期。
